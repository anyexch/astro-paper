### 鼠标动效交互原理：

当然。这个网页的核心思路其实是：**把视频动画变成一组透明序列帧，然后用鼠标位置决定当前显示哪一帧**。

**1. 从视频抽帧**
先把原视频拆成很多张图片帧，比如这次是 121 帧。每一帧都代表小狗某个头部方向或过渡动作。

**2. 抠掉绿幕背景**
对每张图片做绿幕抠像，把绿色背景变成透明。重点是只抠“从画面边缘连通进来的绿色背景”，这样能避免误抠狗身上的白毛、灰毛、眼睛高光等区域。然后再对边缘做一点扩张和去绿边，减少毛发周围的绿色残留。

**3. 合成 sprite 图**
把所有透明帧排进一张大图里，也就是 `sprite.webp`。网页不需要加载 121 张小图，只加载一张 sprite，然后通过 CSS `background-position` 显示其中某一格。

**4. 建立角度到帧号的校准表**
这里最关键。不能简单用：

```js
angle / 360 * frameCount
```

因为视频里的动作不一定是匀速转一圈。我们先看 contact sheet，人工确认“右、下、左、上、左上”等关键方向对应哪一帧，再把关键帧之间的真实视频帧插进去，形成 `ANGLE_KEYS`。

也就是说，它不是数学平均分配，而是：

```js
鼠标方向角度 -> 找最近的校准角度 -> 显示对应真实视频帧
```

**5. 鼠标控制**
网页监听整个页面的 `pointermove`。以屏幕中心为原点，计算鼠标和中心点的夹角：

```js
const angle = (Math.atan2(dy, dx) * 180 / Math.PI + 360) % 360;
```

然后从 `ANGLE_KEYS` 中找最接近这个角度的帧，切换 sprite 位置。

**6. 页面视觉层级**
背景是一层，小狗后面放 logo，小狗本身放在前景。因为小狗帧已经是透明背景，所以可以自然压在 logo 前面。

以后做其他产品也可以套这个流程：

```text
视频/动画素材
-> 抽帧
-> 抠背景/清理主体
-> 合成 sprite
-> 人工校准关键方向或状态
-> 鼠标/滚动/按钮/时间轴驱动帧切换
-> 做成互动网页
```

适合做：吉祥物跟随鼠标、产品 360 度展示、角色表情切换、滚动驱动动画、交互式品牌首页。
